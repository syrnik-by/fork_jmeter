# НТ Мастер

**НТ Мастер** — инструмент нагрузочного и функционального тестирования, разработанный в рамках Банк ПСБ на базе Apache JMeter.

## Что это такое?

НТ Мастер позволяет измерять производительность и проводить нагрузочное тестирование статических и динамических веб-приложений.

Инструмент используется для симуляции высокой нагрузки на сервер, группу серверов или сетевые объекты — с целью проверки устойчивости системы и анализа производительности при различных типах нагрузки.

## Возможности

Полная переносимость, реализация на 100% на Java.

Многопоточность позволяет одновременно выполнять запросы из множества потоков и независимо тестировать разные функции с помощью отдельных групп потоков.

### Поддерживаемые протоколы

- Web — HTTP, HTTPS (Java, NodeJS, PHP, ASP.NET и др.)
- SOAP / REST веб-сервисы
- FTP
- Базы данных через JDBC
- LDAP
- Middleware через JMS
- Почта — SMTP(S), POP3(S), IMAP(S)
- Нативные команды и shell-скрипты
- TCP
- Java-объекты

### IDE

Полнофункциональная среда разработки тестов с поддержкой **записи** (из браузера или нативных приложений), **построения** и **отладки** тест-планов.

### Командная строка

Режим без GUI (headless) для запуска нагрузочных тестов на любой Java-совместимой ОС (Linux, Windows, macOS).

### Отчётность

- Полноценные динамические HTML-отчёты
- Передача метрик в реальном времени в InfluxDB или Graphite (отображение в Grafana)

### Корреляция

Простое извлечение данных из ответов в форматах HTML, JSON, XML и любом текстовом формате.

### Расширяемость

- Подключаемые сэмплеры для неограниченных возможностей тестирования
- Скриптовые сэмплеры (JSR223, Groovy)
- Подключаемые модули статистики и визуализации
- Динамические функции для формирования тестовых данных
- Интеграция с Maven, Gradle и CI/CD-системами

## Требования

- **Java 8** Runtime Environment (JDK предпочтительнее — нужен `keytool` для записи HTTPS)
- Дополнительные JAR (при необходимости, помещаются в папку `lib/`):
  - JDBC-драйвер — от поставщика базы данных
  - JMS-провайдер
  - [Bouncy Castle](https://www.bouncycastle.org/) — только для SMIME Assertion

## Установка

> Пробелы в именах директорий могут вызывать проблемы.

Распакуйте бинарный архив в подходящую директорию.

## Запуск

1. Перейдите в папку `bin/`
2. Запустите `jmeter` (Linux/macOS) или `jmeter.bat` (Windows)

### Windows — дополнительные скрипты

Можно перетащить `.jmx`-файл на один из следующих скриптов:

- `jmeter-n.cmd` — запуск в режиме non-GUI
- `jmeter-n-r.cmd` — запуск в режиме non-GUI с удалённым управлением
- `jmeter-t.cmd` — открыть файл в GUI

## Сборка проекта

### Запуск GUI для разработки

```sh
./gradlew runGui
```

### Сборка дистрибутива (JAR в lib/ и bin/)

```sh
./gradlew createDist
```

> **Внимание:** `createDist` полностью пересоздаёт содержимое `lib/` — кастомные плагины будут удалены.

### Полная сборка с тестами

```sh
./gradlew build
```

Если окружение без GUI:

```sh
./gradlew build -Djava.awt.headless=true
```

### Работа через прокси

Добавьте в `~/.gradle/gradle.properties`:

```properties
systemProp.http.proxyHost=proxy.example.com
systemProp.http.proxyPort=8080
systemProp.http.proxyUser=your_user
systemProp.http.proxyPassword=your_password
systemProp.https.proxyHost=proxy.example.com
systemProp.https.proxyPort=8080
systemProp.https.proxyUser=your_user
systemProp.https.proxyPassword=your_password
```
